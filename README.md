## 系統簡介

### ✨ 主要功能

這是一個現代化的測試案例管理系統，提供：

- 🖥️ **網頁UI介面** - 友善的操作介面，不需手動編輯 Excel
- 📤 **真實檔案上傳** - 截圖和測試資料檔案真正上傳到伺服器
- 📊 **自動生成報告** - 一鍵生成專業的 Excel 測試報告
- 🖼️ **自動嵌入圖片** - 截圖和測試資料圖片自動嵌入 Excel
- 💾 **資料持久化** - 自動儲存到瀏覽器，支援匯出/匯入 JSON
- 🔄 **超連結導航** - Excel 中可點擊超連結查看截圖和測試資料
- ✅ **預期結果欄位** - 清楚記錄測試的預期結果

### 📦 系統組成

```
測試管理系統/
├── 測試管理系統_v2.html     前端網頁介面
├── backend_api.py           後端 API 服務
├── uploads/                 上傳的檔案儲存目錄
└── 使用說明_v3.1.md         本說明文件
```

## 快速開始

### 1️⃣ 安裝 Python 套件

```bash
pip install flask flask-cors openpyxl pillow
```

或使用：
```bash
python -m pip install flask flask-cors openpyxl pillow
```

### 2️⃣ 啟動後端服務

```bash
cd 測試管理系統資料夾
python backend_api.py
```

**成功訊息：**
```
============================================================
🚀 測試管理系統後端 API v3.0 啟動中...
============================================================
📍 URL: http://localhost:5000
📁 上傳目錄: C:\...\uploads

✨ v3.0 新功能:
   ✓ 真實檔案上傳到伺服器
   ✓ 自動嵌入截圖到 Excel
   ✓ 上傳檔案會儲存在 uploads 目錄
============================================================
```

⚠️ **重要：保持這個視窗開啟！**

### 3️⃣ 開啟網頁介面

**方法一：雙擊開啟**
- 找到 `測試管理系統_v2.html`
- 雙擊開啟

**方法二：拖曳開啟**
- 將 `測試管理系統_v2.html` 拖曳到瀏覽器視窗

**方法三：右鍵開啟**
- 右鍵點擊檔案
- 選擇「開啟檔案」→ 選擇瀏覽器

✅ **推薦瀏覽器：** Chrome, Edge, Firefox

## 功能說明

### 🎯 主要功能區域

#### 1. 專案資訊區
位於頁面頂部，包含：
- **專案名稱** ✅ 必填 - 會用於 Excel 檔名
- **測試負責人** ✅ 必填 - 測試團隊負責人
- **測試日期** ✅ 必填 - 預設今天日期
- **測試環境** ✅ 必填 - 測試環境版本資訊

#### 2. 工具列
提供資料管理功能：
- 💾 **儲存資料** - 手動儲存到瀏覽器
- 📂 **載入資料** - 重新載入儲存的資料
- 📤 **匯出 JSON** - 匯出資料為 JSON 檔案備份
- 📥 **匯入 JSON** - 從 JSON 檔案載入資料
- 🗑 **清除所有資料** - 清空所有測試案例和儲存的資料

#### 3. 測試案例清單
顯示所有測試案例卡片，可以：
- 編輯測試案例
- 上傳檔案
- 刪除測試案例（在卡片底部）

#### 4. 新增測試案例按鈕
- 位於所有測試案例下方
- 大按鈕設計，容易點擊
- 點擊後自動新增一個空白測試案例

#### 5. 統計資訊區
顯示測試統計：
- 總測試案例數
- 通過案例數
- 失敗案例數
- 通過率（百分比）

## 檔案上傳

### 📸 執行截圖上傳

**位置：** 測試案例卡片中，執行結果欄位下方

**支援格式：** `.png`, `.jpg`, `.jpeg`, `.gif`, `.bmp`

**操作步驟：**
1. 點擊「點擊上傳執行截圖到伺服器」區域
2. 選擇一張或多張圖片（可按 Ctrl 多選）
3. 點擊「開啟」
4. 等待上傳完成（會顯示上傳進度）

**成功標誌：**
- ✅ 彈出綠色通知「✅ 成功上傳 X 個執行截圖」
- ✅ 上傳區域變成綠色邊框
- ✅ 顯示「✓ 已上傳 X 個檔案」
- ✅ 檔案清單顯示 ✅ 圖示和檔案大小

**檔案儲存位置：**
```
uploads/
└── TC-001/
    └── screenshots/
        ├── screenshot1.png
        └── screenshot2.jpg
```

---

### 📎 測試資料檔案上傳

**位置：** 測試步驟下方，測試資料文字框下方，預期結果欄位上方

**支援格式：** 所有檔案類型

**特殊處理：**
- 🖼️ **圖片格式** (.png, .jpg, .jpeg, .gif, .bmp) → 自動嵌入 Excel
- 📄 **其他格式** (.xlsx, .pdf, .txt等) → 顯示檔案資訊

**操作步驟：**
1. 點擊「點擊上傳測試資料檔案到伺服器」區域
2. 選擇一個或多個檔案
3. 點擊「開啟」
4. 等待上傳完成

**成功標誌：**
- ✅ 彈出綠色通知
- ✅ 上傳區域變成綠色邊框
- ✅ 檔案清單顯示 ✅ 圖示

**檔案儲存位置：**
```
uploads/
└── TC-001/
    └── dataFiles/
        ├── test_data.xlsx
        ├── sample_data.png

```

## Excel 報告

### 📊 生成報告

**步驟：**
1. 確認所有必填欄位已填寫
   - 專案資訊：專案名稱、測試負責人、測試日期、測試環境
   - 測試案例：測試需求、測試個案、測試步驟、預期結果、測試人員
2. 確認後端服務正在運行
3. 點擊「📊 生成測試報告 Excel」按鈕
4. 等待生成（通常 2-5 秒）
5. 自動下載 Excel 檔案

**檔名格式：**
```
專案名稱_日期_時間.xlsx

範例：
系統功能測試_20260113_143025.xlsx
登入功能測試_20260113_150815.xlsx
```

|------|---------|---------|---------|---------|---------|---------|--------|---------|---------|---------|
| TC-001 | 登入功能 | 驗證正確帳密 | ... | ... | 檔案1,2 | 顯示登入成功 | 高 | 通過 | 張三 | 截圖1,2 |

## 資料管理

### 💾 自動儲存

## 常見問題

### ❓ Q1: 為什麼需要「預期結果」欄位？

**答：** 
預期結果欄位用來明確記錄測試案例的預期結果，這是測試案例的重要組成部分：

1. **清楚定義成功標準** - 明確什麼情況下測試算通過
2. **便於驗證** - 執行測試時可以對照預期結果
3. **提高可讀性** - 閱讀報告的人可以快速了解測試目的
4. **符合測試規範** - 完整的測試案例應包含預期結果

**範例：**
```
測試步驟:
1. 開啟登入頁面
2. 輸入正確帳號密碼
3. 點擊登入按鈕

預期結果:
• 系統顯示「登入成功」訊息
• 自動導向主頁面
• 主頁面顯示使用者名稱
• 可以看到功能選單
```

---

### ❓ Q2: 點擊「新增測試案例」沒反應？

**可能原因：**
1. 瀏覽器快取問題
2. JavaScript 錯誤
3. 舊資料相容性問題

**解決方法：**

**方法 1：強制重新整理**
```
按 Ctrl + F5 (Windows)
或 Cmd + Shift + R (Mac)
```

**方法 2：清除資料**

3. 重新建立測試案例（包含預期結果）

## 疑難排解

### 🔧 後端無法啟動

**症狀：**
```
ModuleNotFoundError: No module named 'flask'
```

**解決方法：**
```bash
pip install flask flask-cors openpyxl pillow
```

---

**症狀：**
```
Address already in use
```

**解決方法：**
1. 關閉其他佔用 5000 埠的程式
2. 或修改後端埠號：
```python
app.run(debug=True, host='0.0.0.0', port=5001)  # 改成 5001
```

前端也要修改：
```javascript
const API_URL = 'http://localhost:5001';  # 改成 5001
```

---

### 🔧 檔案上傳失敗

**診斷步驟：**

**1. 檢查後端 Console**
```bash
# 成功訊息
✅ 檔案已上傳: uploads/TC-001/screenshots/test.png

❌ 上傳檔案錯誤: [錯誤內容]
```

**2. 檢查檔案權限**
確保 `uploads` 目錄有寫入權限

**3. 檢查檔案大小**

**4. 檢查防火牆**
確認防火牆沒有阻擋 Python

**診斷步驟：**

```bash
# 查看詳細錯誤訊息
```

**2. 檢查圖片檔案**

## 💡 使用技巧

### 技巧 1: 填寫預期結果的最佳實踐

**好的預期結果範例：**
```
✅ 明確、可驗證：
• 系統顯示「登入成功」訊息
• 頁面導向至 /dashboard
• 顯示使用者名稱「張三」
• 登入按鈕變為「登出」
```

**不好的預期結果範例：**
```
❌ 模糊、不具體：
• 登入成功
• 正常運作
• 符合需求
```

**撰寫技巧：**
1. 使用項目符號列出多個預期結果
2. 具體描述可觀察的現象
3. 包含畫面、訊息、資料的預期狀態
4. 避免使用模糊的詞彙

---

### 技巧 2: 批量複製測試案例

1. 匯出現有測試案例為 JSON
2. 用文字編輯器開啟 JSON
3. 複製測試案例物件
4. 修改 ID、內容和預期結果
5. 匯入修改後的 JSON

---

### 技巧 3: 團隊協作

1. 測試人員 A 建立測試案例（含預期結果）→ 匯出 JSON
2. 測試人員 B 匯入 JSON → 執行測試 → 上傳截圖
3. 測試人員 C 匯入 JSON → 生成最終報告

---

### 技巧 4: 定期備份

建議每週或每完成一個測試階段就：
1. 匯出 JSON 備份
2. 生成 Excel 報告
3. 備份 `uploads` 目錄

---

### 技巧 5: 測試案例撰寫順序

推薦的填寫順序：
1. 測試需求、測試人員
2. 測試個案
3. 前置條件
4. 測試步驟
5. 測試資料
6. **預期結果**（根據測試步驟填寫）
7. 上傳檔案
8. 執行測試，選擇結果
